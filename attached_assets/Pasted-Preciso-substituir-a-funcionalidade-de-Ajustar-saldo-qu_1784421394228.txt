create table if not exists balance_snapshots (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  bank_id uuid not null references banks(id) on delete cascade,
  snapshot_date date not null,
  balance numeric not null,
  created_at timestamptz not null default now()
);
create index if not exists idx_balance_snapshots_bank on balance_snapshots(bank_id, snapshot_date desc);
create index if not exists idx_balance_snapshots_user on balance_snapshots(user_id);
alter table balance_snapshots enable row level security;
-- adicione as policies de RLS no mesmo padrão das outras tabelas do projeto (select/insert/update/delete restritos a auth.uid() = user_id)

2. MUDAR O CÁLCULO DE SALDO DA CONTA
- Em src/services/dataService.ts (ou onde quer que o saldo de cada conta seja calculado hoje), mude a lógica para, por conta bancária:
  a) Buscar o snapshot mais recente daquela conta (maior snapshot_date, e entre snapshots do mesmo dia o mais recente por created_at).
  b) Se existir snapshot: saldo atual = snapshot.balance + soma das transações vinculadas àquela conta com date > snapshot.snapshot_date (mais transferências de/para essa conta no mesmo período).
  c) Se NÃO existir snapshot: mantenha o cálculo antigo (initial_balance + soma de todas as transações da conta), exatamente como funciona hoje.
- Aplique essa mesma lógica em TODOS os lugares que hoje calculam saldo de conta: Dashboard (cards de Contas Bancárias e Saldo total), Configurações > Bancos, e no próprio fluxo de "Ajustar saldo" (para mostrar o saldo atual antes do ajuste).

3. ATUALIZAR O GRÁFICO "EVOLUÇÃO DO SALDO" (Relatórios)
- Ajuste o cálculo dia a dia para, em cada dia D: usar o snapshot mais recente com snapshot_date <= D como base (se existir), somando as transações entre o snapshot e o dia D; caso não haja snapshot antes de D, usar o cálculo antigo (initial_balance + transações até D). Isso garante que dias ANTES de um ajuste continuem mostrando o saldo histórico real, e dias DEPOIS reflitam o novo marco.
